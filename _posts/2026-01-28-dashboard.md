Nach mehreren internen Entwicklungsschleifen, Diskussionen zur Darstellbarkeit und Anpassungen im Backend unseres digitalen Zwillings freuen wir uns, dass die erste Version unseres Dashboards nun von Mitarbeitenden der Verkehrsgesellschaft Frankfurt am Main mbh (VGF) getestet wird.

![dashboard_einzelmessung](/assets/img/dashboard_einzelmessung.jpg){: w="500" h="400" }{: w="500" h="400" }
_Abb. 1: Messwerte von Einzelmessungen_
Im Dashboard stehen einzelne Seiten zu Häufigkeit der Messungen, Analyse der rohen Messwerte von Einzelmessungen (Abb. 1) und Ergebnisdarstellungen zum Längshöhenfehler zur Verfügung. Das Dashboard auf Basis der Open-Source-Anwendung Grafana ermöglicht die grafische Analyse der Messungen aus den im VGF-Netz verkehrenden Straßen- und Stadtbahnen.

![dashboard_teppich](/assets/img/dashboard_teppich.jpg){: w="500" h="400" }{: w="500" h="400" }
_Abb. 2: Teppichdarstellung zur Fehlerentwicklung des Längshöhenfehlers_
Mit einer linienbasierten Übersicht zum Streckenzustand als Kartendarstellung lassen sich aktuelle Probleme im Streckennetz schnell erkennen. Eine tiefere Analyse ermöglichen Teppichdiagramme zur Darstellung der Fehlerentwicklung über die vergangenen Messwochen (Abb 2). Dies wird durch das kontinuierliche Monitoring der Strecken und eine Nachbearbeitung der Messdaten im Backend des digitalen Zwillings ermöglicht. Weiterhin lassen sich so Trends in der Fehlerentwicklung darstellen. Dies erfolgt aktuell in einer grafischen Kartendarstellung mit Hilfe von farbkodierten Pfeilen (Abb. 3).

![dashboard_trend](/assets/img/dashboard_trend.jpg){: w="500" h="400" }{: w="500" h="400" }
_Abb. 3: Kartendarstellung mit Trends_
Das Dashboard und die einzelnen Darstellungen werden nun im Austausch mit den Mitarbeitenden der VGF und dem Projektteam analysiert und weiterentwickelt.
